# Karaoke TV 播放器 UI 开发指引

> 设计源文件：`design.pen`（Pencil 编辑器打开）  
> 后端 API：`../app/ARCHITECTURE.md`  
> 本文档描述 **Android TV 播放端 Wrapper** 的视觉规范、页面结构、遥控器交互与 API 映射。

---

## 0. 产品定位

TV 端是 **KTV 大屏播放器 + 轻量遥控壳**，不是完整点歌台：

| 职责 | TV 端 | 手机 / Web RC |
|------|-------|---------------|
| 全屏视频播放 | ✅ | ❌ |
| 曲库浏览点歌 | 侧边栏简化版 | ✅ 主流程 |
| 队列维护 | 侧边栏（置顶/删除） | ✅ |
| 曲库导入/编辑 | ❌ | ❌（Admin） |

**设计原则：** 主界面 90% 留给播放画面；点歌/队列/设置通过 **右侧滑出面板** 完成；所有操作适配 **D-pad 焦点导航**。

### 播放器优先（核心）

| 原则 | 说明 |
|------|------|
| **PlayerView 为主体** | 进入播放页后全屏 `PlayerView` 始终垫底；播放中不绘制应用背景、圆角卡片或装饰性渐变 |
| **辅助控件极简** | 顶栏 / 菜单提示 / 侧边栏均为半透明叠层，5s 无操作自动淡出，不遮挡画面主体 |
| **空内容态** | 无曲目时：黑屏（`#000000`）+ 半透明遮罩 + 居中提示文案；**不用**圆角卡片充当「播放背景」 |
| **中间态** | 准备中 / 连接失败：全屏遮罩 + 文案 / 进度条；**不用**卡片面板盖住播放器区域 |
| **仅配置页可用卡片** | `首次连接`、侧边栏面板等**非播放区域**可使用 `bg-elevated` 卡片与圆角 |
| **编解码** | ExoPlayer 使用 `DefaultRenderersFactory`，优先硬件解码 + 解码器回退 |

## 1. 设计稿结构

画布比例 **1920×1080**（16:9 TV，设计基准）。实现层通过 `KaraokeScaleProvider` 按实际屏幕等比缩放（`min(屏宽/1920, 屏高/1080)`）。Frame 名称前缀：

| 前缀 | 含义 |
|------|------|
| `提示 ·` | 分区说明、交互规范 |
| `页面 ·` | 完整屏幕 |
| `状态 ·` | 叠加在主播放页上的中间态 |

### 页面一览

| 设计 Frame | 说明 | 进入方式 |
|------------|------|----------|
| `页面 · 首次连接` | 无已保存地址时展示 | 冷启动 |
| `页面 · 主播放页` | 空闲 / 等待点歌 | 已连接默认页 |
| `页面 · 主播放页 · 播放中` | 正在播放 + 顶栏信息 | 队首 `playing` |
| `页面 · 侧边栏 · 点歌` | 曲库搜索 + 列表点歌 | MENU / 右键打开 |
| `页面 · 侧边栏 · 已点` | 已点 Tab · **浏览态**（仅查看，无按钮） | 侧边栏 Tab 切换 |
| `页面 · 侧边栏 · 已点 · 操作态` | 已点 Tab · **操作态**（置顶/移除，默认置顶） | 浏览态对歌曲 OK |
| `页面 · 侧边栏 · 设置` | 修改服务器地址 | 侧边栏 Tab 切换 |
| `状态 · 准备中` | 资源转码/缓存遮罩 | `ensure-ready` 等待 |
| `状态 · 连接失败` | 无法连接后端 | 启动 / 设置测试失败 |

## 2. 启动与地址持久化

```
冷启动
  ├─ SharedPreferences 无 server → 页面 · 首次连接
  │     └─ 输入 http://host:15233 → 测试连接 → 保存 → 主播放页
  └─ 已有 server → 直接连接 SSE + 拉队列 → 主播放页
```

- 存储键：`KTV_PREFS.server`（与现版一致）
- **设置页**可随时修改地址；保存后重连 SSE、刷新队列
- 地址格式：`http://192.168.x.x:15233`，末尾不带 `/`

## 3. 遥控器交互

### 3.1 主播放页（侧边栏关闭）

| 按键 | 行为 |
|------|------|
| **MENU** / 长按 OK | 打开侧边栏（默认 Tab：点歌） |
| **OK** / CENTER | 播放/暂停（有曲目时） |
| **◀ 后退** | 无操作（防误触退出 App） |
| **▲▼** | 无操作 |

### 3.2 侧边栏打开

| 按键 | 行为 |
|------|------|
| **◀ 后退** / BACK | 关闭侧边栏，焦点回播放页 |
| **▲** | 内容上移；在首项时回到 Tab 行 |
| **▼** | 从 Tab 行进入内容，或在内容中下移 |
| **◀▶** | 切换 Tab（在 Tab 行或任意内容项上均可） |
| **OK** | Tab 行：进入内容；内容：执行当前项 |

#### 已点 Tab 两阶段交互

| 阶段 | 说明 | 按键 |
|------|------|------|
| **浏览态** | 仅查看队列，行内**不显示**置顶/移除按钮 | ▲▼ 切换歌曲 · OK 进入操作 · ◀ 关闭 |
| **操作态** | 对当前歌曲 OK 后展开操作栏 | ◀▶ 切换置顶/移除 · OK 确认 · ◀ 返回浏览 |

- 进入操作态后默认焦点在 **置顶**（紫色高亮），移除为次要态（灰色），避免误删
- 播放中歌曲（队首）在浏览态仅展示状态，不可进入操作态（或操作态禁用置顶）

### 3.3 首次连接 / 设置

| 按键 | 行为 |
|------|------|
| **▲▼** | 输入框 ↔ 按钮 |
| **OK** | 激活按钮 / 进入输入框 |
| **◀ 后退** | 设置页→关闭侧边栏；首次连接→退出 App |

### 3.4 焦点样式

- 可聚焦项：`3–5px` `$accent-primary` 描边 + `$bg-hover` 背景（内容项用加粗描边）
- 当前 Tab：`$accent-primary` 描边；焦点在 Tab 行时加粗描边
- 底部常驻提示条显示当前上下文可用按键

### 3.5 触摸 / 鼠标（与遥控器并行）

| 区域 | 触摸行为 |
|------|----------|
| **主播放页** | 点击画面：有曲目时播放/暂停，空闲时打开菜单 |
| **主播放页** | 右下角「菜单」按钮：打开侧边栏 |
| **空闲遮罩** | 点击任意处打开点歌菜单 |
| **侧边栏遮罩** | 点击左侧半透明区域关闭侧边栏 |
| **点歌 Tab** | 点击歌曲行点歌；点击「加载更多」 |
| **已点 Tab** | 点击歌曲进入操作态；点击「置顶」「移除」执行；再次点击同一行取消 |
| **设置 Tab** | 点击输入框编辑；点击按钮测试/保存 |
| **首次连接** | 点击输入框、连接按钮（系统软键盘） |

触摸与 D-pad 共用同一套 ViewModel 逻辑，不依赖物理按键。

## 4. 页面规格

### 4.1 主播放页

- **底层：** 全屏 `PlayerView`（无默认控制器、黑场快门、`RESIZE_MODE_FIT`）
- **禁止：** 在播放页根容器使用 `bg-primary` 或圆角背景（会遮挡/污染视频区域）
- **顶栏（播放中显示，5s 无操作淡出）：**
  - 左：当前歌曲名（24sp 粗体）
  - 右：下一首预览 / 连接状态绿点
  - 样式：全宽半透明条（`#00000099`），非圆角卡片
- **空闲：** 黑屏 + 半透明遮罩 + 居中「等待点歌」文案（无圆角卡片、无 emoji 装饰块）
- **右下角：** 纯文字「MENU 菜单」提示（无圆角底）；遥控 `MENU` 打开侧边栏

### 4.2 侧边栏（宽 630px @1920，右侧滑入）

- 左侧 60% 半透明遮罩，点击/后退关闭
- 面板：`$bg-secondary`，左描边 `$border-subtle`
- **顶栏 Tab：** `点歌` | `已点` | `设置`（横向，焦点可切换）
- **点歌 Tab：**
  - 搜索框（软键盘 / 系统 IME）
  - 曲库列表：`歌名` + `+ 点歌` 按钮行，焦点在行上
  - 分页：滚动到底加载更多
- **已点 Tab（浏览态）：**
  - 队列列表：序号 + 歌名 + 状态（播放中/等待）
  - 焦点行仅高亮边框，**无操作按钮**
  - OK 进入该行的操作态
- **已点 Tab（操作态）：**
  - 展开操作栏：`置顶` | `移除`，◀▶ 切换
  - **默认选中置顶**（accent 实心）；移除为灰色次要态
  - OK 执行当前操作后返回浏览态；◀ 取消返回浏览态
- **设置 Tab：**
  - 服务器地址输入
  - `测试连接` `保存并重连`
  - 当前地址只读展示

### 4.3 首次连接

- 居中卡片（宽 840px @1920）
- 标题 + 说明 + 地址输入 + `连接` 主按钮
- 无侧边栏、无顶栏

## 5. API 映射

| UI 行为 | API |
|---------|-----|
| 测试/保存连接 | `GET /api/v1/library/songs?page=1&q=` |
| 曲库列表 | `GET /api/v1/library/songs?page=&q=` |
| 点歌 | `POST /api/v1/queue/songs/{id}` |
| 队列列表 | `GET /api/v1/queue` |
| 置顶 | `POST /api/v1/queue/songs/{id}/top` |
| 移除 | `DELETE /api/v1/queue/songs/{id}` |
| 播放配置 | `GET /api/v1/playback/songs/{id}` |
| 准备状态 | `GET /api/v1/playback/songs/{id}/prepare-status` |
| 确保就绪 | `POST /api/v1/playback/songs/{id}/ensure-ready` |
| 媒体流 | `GET /api/v1/playback/stream/{id}/{kind}` |
| 标记演唱中 | `POST /api/v1/playback/session/singing/{id}` |
| 标记完成 | `POST /api/v1/playback/session/finished/{id}` |
| SSE | `GET /api/v1/events` |

## 6. Design Tokens

与 RC 点歌台共用色系（`design.pen` variables）：

| Token | 值 | TV 用途 |
|-------|-----|---------|
| `bg-primary` | `#0B0B12` | 播放背景、遮罩底 |
| `bg-secondary` | `#14141F` | 侧边栏面板 |
| `bg-elevated` | `#1C1C2E` | 卡片、输入框 |
| `accent-primary` | `#A855F7` | 焦点环、主按钮 |
| `text-primary` | `#F5F5F7` | 歌名、标题 |
| `text-secondary` | `#9494A8` | 副文案 |
| `connected` | `#69F0AE` | 连接状态点 |

**TV 字号建议（1920×1080 设计稿，运行时自适应）：** 标题 36sp / 列表 30sp / 正文 24sp / 遥控提示 21sp

实现：`DesignTokens` 存基准值 → `wdp()` / `ssp()` 按屏幕缩放。
